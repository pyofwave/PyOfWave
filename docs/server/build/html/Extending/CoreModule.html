Using PyOfWave’s Core module¶

Note

Not yet fully implemented.

pyofwave_server.core provides facilities that are shared, in various methods, by all protocols and extensions. Specifics for extending this are provided by Integrating Your Own Authentication Scheme, Data Storage Extensions, Protocol Operators, and Adding New Protocols to the PyOfWave Server.

core.datasource¶

The interface DataSource provides access to persistant XML following the wave schema. LXML is used to represent the XML files.

newDocument()¶

Creates a new document in the DataSource. Returns the blank lxml.etree.ElementBase object if successful.

getDocument(url)¶

Loads the document from the DataSource. Returns a lxml.etree.ElementBase object.

getDocumentVersion(url, start, end, limit)¶

Loads the document at the specified times. Returns a iterable of lxml.etree.ElementBase objects.

searchDocuments(user, search)¶

Returns a list of wave URLs to documents that match the search in the form of an xQuery conditional String object.

core.delta¶

core.delta, provides a classic Observer interface. This module is used to distribute changes to documents, “deltas”, generated by the operations in the Wave Protocol. These deltas can be in either “alpha” or “beta” state (after and before being commited to storage).

Any one has the ability to distribute deltas on a DeltaObservable, but you should only do it on BetaDeltaObservable leaving DataSource objects to distribute on AlphaDeltaObservable, so as that the Deltas distributed there are truly in alpha state.

Use the applyDelta() function to apply a delta to XML. This is mostly for persistance extensions.

applyDelta(doc, delta)¶

Combines the LXML eTree elements passed to it. Elements with a delete attribute are deleted.

DeltaObservable¶

DeltaObservable distributes Delta objects throughout all extensions.

class DeltaObservable¶

Distributes deltas between different components of PyOfWave server.

applyDelta(doc, delta)¶

Notifies all observers that doc has/will (depending on object) had the changes in delta applied.

Delta is a partial LXML tree with delete properties marking elements to remove.

addObserver(observer, url = "")¶

Calls observer when applyDelta() is called, passing the same arguments.

removeObserver(observer)¶

Stops calling observer.

There’s two instances of this class you’ll use to distribute Delta objects:

AlphaDeltaObservable¶

DataSource objects pushes deltas through this DeltaObservable, which indicates that all Delta objects from this object has been commited to storage.

BetaDeltaObservable¶

DataSource objects are observers of this object, which means that by sending Delta objects on this object, you are commiting them to memory.

core.operation¶

core.operation has only one function, which imports a function from pyofwave_server.operations and executes it with given arguments.

performOperation(ip, operation, kwargs)¶

Executes the operation operation and provides it ip and kwargs. It will either raise a OperationError providing a status dictionary and code (as in HTTP Error codes), or return a mapping.

class Events¶

Handles notification of selected events.

__init__(authzid, callback)¶

Initiates an object for user identified by authzid, and calls callback to notify the caller of new events.

register(url, event)¶

Starts notifying the callback when event happens in document identified by url. event is either an Operation or a String.

unregister(url, event = "*")¶

Stops notifying the callback when event happens in document identified by url.

authzid¶

The Python SASL authorization ID.

Instructions for adding operations are in Protocol Operators.
